## Purpose

Keep a multi-agent team coherent after the exciting setup phase ends.

## Procedure

1. Create a team registry with profile owner, mission, tools, memory scope, and policy ceiling.
2. Define handoff contracts between common pairs such as orchestrator to researcher, researcher to writer, writer to engineer.
3. Schedule memory audits: remove stale facts, merge duplicates, and flag contradictions.
4. Check for drift: profiles doing work outside their domain.
5. Check for rot: old instructions or broken tools still trusted.
6. Check for bloat: too many files loaded into runtime context.
7. Check for collision: two agents owning the same task or writing conflicting state.
8. Produce a monthly operator report with fixes and deprecations.

## Outputs

- Team registry
- Handoff contracts
- Memory audit report
- Operator maintenance checklist

## Validation

- Every profile has a policy ceiling.
- Monthly report identifies at least drift/rot/bloat/collision status.
- Deprecated roles or files are archived rather than silently left active.
