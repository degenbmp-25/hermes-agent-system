# obsidian-unlimited-context

## Source Provenance

This is a BeastmodeVault-authored operational skill derived from: https://x.com/dallasaptgp/status/2033574400901935534

Use the source as provenance and background only. Follow the procedure below as the executable workflow.

## Purpose

Use local linked files as the durable context substrate for agents instead of repeatedly uploading scattered documents.

## Procedure

1. Define the vault scope: personal, company, client, or project. Avoid mixing sensitive domains by default.
2. Create a simple folder taxonomy and naming rules before importing large piles of files.
3. Move source documents into references or source folders without rewriting them first.
4. Create index/context-pack notes that summarize and link to source material.
5. Use wikilinks and source manifests for traceability.
6. Give agents progressive-discovery instructions: search first, read only relevant files, cite paths.
7. Run validation for links, manifest entries, and secrets before treating the vault as production context.

## Outputs

- Vault taxonomy
- Context-pack notes
- Source import plan
- Agent discovery rules

## Validation

- A new agent can find the right context without loading the whole vault.
- Sensitive domains are separated.
- Important claims trace back to source files.
